# 実測実施計画書 — SaunaFlow CFD バリデーション

**作成日**: 2026-04-22  
**目的**: CFD (Case L-1) の予測値を実測で検証し、モデルの信頼性を定量評価する

| プローブ | 位置 (x, y, z) | L-1 予測 (°C) | L-2 予測 (°C) | 許容差 |
|---------|---------------|-------------|-------------|--------|
| upper_bench | (1.5, 2.0, 1.25) | 95 | TBD | ±5°C |
| lower_bench | (1.5, 0.8, 1.25) | 54 | TBD | ±5°C |

## 2. 計測システム構成

### 2.1 ハードウェア（既存）

```
[サウナ室内]                    [サウナ室外]
                                
DHT22 ──────────┐              
(sauna air)     │              
                ├─ Pico W/ESP32 ──USB──→ PC (serial_logger.py)
DS18B20 ────────┘              
(box internal)                  
```

| コンポーネント | 仕様 | 用途 | 制限 |
|-------------|------|------|------|
| DHT22 | -40~80°C, ±0.5°C, 2s間隔 | サウナ空気温度・湿度 | **上限80°C** |
| DS18B20 | -55~125°C, ±0.5°C | 筐体内部温度（安全監視） | — |
| Pico W / ESP32 | MicroPython | データ取得・シリアル送信 | — |

### 2.2 センサー限界と対策

**重大な制約**: DHT22 の上限は **80°C**。L-1 の upper_bench 予測は **95°C**。

| 位置 | 予測温度 | DHT22 対応可否 | 対策 |
|------|---------|--------------|------|
| upper_bench (y=2.0m) | 95°C | **不可** (仕様超過) | K型熱電対 + MAX31855 が必要 |
| lower_bench (y=0.8m) | 54°C | **可** | DHT22 で計測可能 |
| floor_level (y=0.1m) | 31°C | **可** | DHT22 で計測可能 |

**Phase 1 の現実的な計測対象**: lower_bench と floor_level の 2 点。
upper_bench は Phase 2 (K 型熱電対導入後) で計測。

## 3. 計測ポイント設計

### 3.1 CFD プローブ位置との対応

```
              ceiling (y=2.5m)
    ┌──────────────────────────────┐
    │                              │
    │    ★ upper_bench             │  y=2.0m  ← Phase 2 (K型)
    │       (1.5, 2.0, 1.25)      │
    │                              │
    │    ● lower_bench             │  y=0.8m  ← Phase 1 (DHT22)
    │       (1.5, 0.8, 1.25)      │
    │                              │
    │    ○ floor_level             │  y=0.1m  ← Phase 1 (DHT22)
    │       (1.5, 0.1, 1.25)      │
    └──────────────────────────────┘
              floor (y=0.0m)
    
    x=1.5m (部屋中央), z=1.25m (部屋中央)
```

### 3.2 設置の実用上の注意

| 項目 | 要件 | 理由 |
|------|------|------|
| 位置精度 | ±10cm | CFD のセルサイズ (M0) が ~30cm なので ±10cm で十分 |
| 壁面からの距離 | ≥30cm | 壁面境界層の影響を避ける |
| ヒーターからの距離 | ≥50cm | 直接放射の影響を避ける |
| 気流の遮蔽 | なし | センサーを大きな構造物で覆わない |
| 配線 | 耐熱被覆 | 80°C 環境でケーブルが劣化しないこと |

### 3.3 設置方法

1. **lower_bench (y=0.8m)**: 下段ベンチの木材表面から 5cm 上に設置
   - 木製スタンドまたはクリップで固定
   - センサー先端が空気中に露出（木材接触させない）

2. **floor_level (y=0.1m)**: 床面から 10cm の高さに設置
   - 小さな三脚またはブロックの上
   - 部屋中央付近 (x≈1.5m, z≈1.25m)

## 4. 計測手順

### 4.1 事前準備（計測日の前日まで）

| # | 作業 | 所要時間 | 完了判定 |
|---|------|---------|---------|
| 1 | センサー校正（氷水点 0°C） | 30分 | オフセット ±1°C 以内 |
| 2 | 相互校正（2個の DHT22 を同一環境で比較） | 15分 | 個体差を meta.yaml に記録 |
| 3 | ファームウェア書き込み・動作確認 | 15分 | JSON 出力がシリアルに流れる |
| 4 | serial_logger.py 接続確認 | 10分 | CSV ファイルに記録される |
| 5 | 耐熱筐体準備 | — | DS18B20 が筐体内温度を監視 |
| 6 | 配線の耐熱確認 | — | 80°C 環境で 30 分以上保持 |

### 4.2 計測当日プロトコル

```
時刻          作業
────────────────────────────────────────────
T-60min      サウナ室ヒーター ON (予熱開始)
             → 実際のサウナの予熱時間に合わせる

T-10min      計測機器を室外で起動
             serial_logger.py 開始
             室温でのベースライン記録（約5分間）

T-5min       サウナ室温をサウナ温度計で確認
             → 80°C 以上なら定常到達と推定
             筐体温度 (DS18B20) を確認 < 30°C

T=0          センサーをサウナ室に投入
             lower_bench 位置に設置
             floor_level 位置に設置
             ドア閉鎖、タイマー開始

T+0~10min    [過渡区間] センサーが室温から昇温中
             dT/dt を監視
             筐体温度 (DS18B20) を監視 → 50°C で WARN

T+10min      定常到達判定: |dT/dt| < 0.1°C/min for 60秒
             → 到達したら「定常開始」をメモ

T+10~30min   [定常区間] 最低 15 分の連続記録
             途中でドアを開けない
             人の出入りなし

T+30min      記録停止
             センサー撤収（素手で触らない、熱い）
             筐体温度を確認 → 60°C 未満であること

T+35min      CSV ファイルの確認
             サンプル数確認: ~900 行 (30min × 30/min)
             欠損データ (-999) が全体の 5% 未満であること
```

## 6. 判定基準

### 6.1 成功判定

| 指標 | 基準 | 根拠 |
|------|------|------|
| lower_bench 絶対誤差 | ≤ ±10°C | M0 mesh + 簡易壁モデルの精度限界 |
| floor_level 絶対誤差 | ≤ ±5°C | 低温域はモデル精度が高い |
| 成層化方向 | CFD と同じ (upper > lower > floor) | 定性的一致 |
| 再現性 | セッション間 ±2°C | 計測の信頼性 |

### 6.2 結果の解釈シナリオ

| シナリオ | CFD vs 実測 | 解釈 | 次のアクション |
|---------|-----------|------|-------------|
| A: 全点 ±5°C | 良好な一致 | モデル妥当 | Phase 2 へ進む |
| B: lower OK, floor ずれ | 壁面モデルの問題 | 床材 BC の修正 |
| C: 全点 +10°C 以上 | CFD が高温すぎ | volume source 低減 or 壁面損失増加 |
| D: 全点 -10°C 以上 | CFD が低温すぎ | volume source 増加 or 壁面損失低減 |
| E: 方向が逆 | モデル根本問題 | ヒーター位置/換気 BC の見直し |

## 9. リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| DHT22 が 80°C 環境で故障 | 計測不可 | lower_bench (54°C) のみに限定、筐体で保護 |
| USB ケーブルの耐熱不足 | 通信切断 | マイコンをドア外に置き、センサーのみ延長 |
| ヒーターの ON/OFF サイクル | 温度変動 | 定常判定に移動平均を使用 |
| サウナ室の仕様がモデルと異なる | 比較困難 | 実測時に room dimensions を確認・記録 |
| 配線延長による DHT22 信号劣化 | 読み取りエラー | 延長 3m 以内、プルアップ抵抗 4.7kΩ |

## 10. CFD 側の準備

実測前に CFD 側で準備すべきこと:

1. **実際のサウナ室寸法を入力** — 現在の 3.0×2.5×2.5m を実測値に更新
2. **ヒーター出力の確認** — 実機の定格 (kW) を確認
3. **壁材の確認** — 木材の種類・厚み → wall_htc の見直し
4. **換気の確認** — 給排気口の有無・サイズ・位置
5. **L-1 相当の transient を再実行** — 実寸法で 3600s

---

**この計画書の要点**: DHT22 で lower_bench (54°C) と floor_level (31°C) を計測し、CFD L-1 の予測と比較する。upper_bench (95°C) は DHT22 の仕様外なので Phase 2 (K 型熱電対) で対応。まず 1 点でも実測を取ることが最優先。
